# mgate-agent v0.1.0-rc1

## 版本定位

这是 `mgate-agent` 的第一个 RC 版本，用于随身 WiFi Debian 设备的远程安全执行链路验证。

本版本目标是验证：

```text
transport 收消息
  -> commands.Handler
    -> action registry
    -> runner
      -> mgate.sh
  -> result outbox
  -> transport 回传
```

## 已支持

- YAML 主配置：`/etc/mgate-agent/agent.yaml`
- JSON credentials：`/var/lib/mgate-agent/credentials.json`
- WebSocket 主通道
- HTTPS Pull 兜底通道
- WebSocket / Pull HMAC header 鉴权
- 白名单 action registry
- 本地 `security.allow_actions` 二次收窄
- `commands.Handler` 安全门禁
- command_id 去重
- 安全 runner：固定 `mgate.sh` + argv 数组
- result outbox 持久化补发
- check / doctor
- fake cloud smoke test
- systemd 部署
- install / uninstall 脚本
- Linux amd64 / arm64 / armv7 交叉编译
- release tar.gz target
- `checksums.txt` SHA256 校验文件
- dev/main 分支 GitHub Release 自动发布 workflow

## 安全边界

- 不支持任意 shell。
- cloud 不能下发任意命令行。
- transport 不直接调用 runner。
- 所有 command 必须进入 `commands.Handler`。
- 所有本地动作必须映射到硬编码 action。
- runner 只调用 `exec.CommandContext(ctx, mgatePath, argv...)`。
- outbox 只保存 result，不保存 command。
- result 发送失败不会触发 command 重放。
- secret/signature 不进入 URL。
- doctor、日志、audit 不输出 `device_secret`。

## 当前不支持

- enroll。
- mgate-cloud 服务端。
- `result_ack` 强确认。
- 新增远程 action。
- AP/TProxy/wlan/mihomo 业务逻辑实现。
- command 持久化或 command 重放。
- MQTT、Redis、SQLite 或第三方队列。

## 已知限制

- WebSocket result 写入成功即认为发送成功，暂未实现 server `result_ack`。
- outbox 是轻量补发机制，不是无限队列。
- outbox 超过 100 条或 5MB 时会丢弃最旧 result。
- result stdout/stderr 会做长度限制和基础脱敏，但不是完整 DLP 系统。
- systemd hardening 采用保守策略，后续会根据 `mgate.sh` 真实访问路径继续收紧。
- v0.1.0-rc1 应优先连接测试 cloud 或 fake cloud 验证。

## 验证方式

```sh
go test ./...
go vet ./...
go build -o bin/mgate-agent ./cmd/mgate-agent
GOOS=linux GOARCH=amd64 go build -o bin/mgate-agent-linux-amd64 ./cmd/mgate-agent
GOOS=linux GOARCH=arm64 go build -o bin/mgate-agent-linux-arm64 ./cmd/mgate-agent
GOOS=linux GOARCH=arm GOARM=7 go build -o bin/mgate-agent-linux-armv7 ./cmd/mgate-agent
```

端到端 smoke test：

```sh
go test ./internal/integration -count=1
```

## 安装说明

推荐使用 release tar.gz：

```sh
tar -xzf mgate-agent-v0.1.0-rc1-linux-armv7.tar.gz
cd mgate-agent-v0.1.0-rc1-linux-armv7
sh scripts/install.sh
```

Release asset 命名规则：

```text
mgate-agent-<tag>-linux-amd64.tar.gz
mgate-agent-<tag>-linux-arm64.tar.gz
mgate-agent-<tag>-linux-armv7.tar.gz
checksums.txt
```

v0.1.0-rc1 示例：

```text
mgate-agent-v0.1.0-rc1-linux-amd64.tar.gz
mgate-agent-v0.1.0-rc1-linux-arm64.tar.gz
mgate-agent-v0.1.0-rc1-linux-armv7.tar.gz
checksums.txt
```

下载后应先校验：

```sh
cd dist
sha256sum -c checksums.txt
```

`dev` 分支用于验证，`main` 分支用于发布。代码进入 `main` 后，`Main Release` workflow 会读取 `VERSION`，先完成 gofmt、vet、test、host build、Linux 构建、release 打包、结构校验和 checksum 校验，再创建 tag、创建 GitHub Release，并上传三个 tar.gz 与 `checksums.txt`。

`v0.1.0-rc1` 会因为包含 `-rc` 自动标记为 pre-release。正式版例如 `v0.1.0` 不会标记 pre-release。如果同名 tag 或 Release 已存在，workflow 会失败，不会自动覆盖旧资产；需要重发 RC 时应更新 `VERSION`，例如 `v0.1.0-rc2`。

安装后准备：

```sh
/etc/mgate-agent/agent.yaml
/var/lib/mgate-agent/credentials.json
```

启动前先执行：

```sh
mgate-agent check --config /etc/mgate-agent/agent.yaml
mgate-agent doctor --config /etc/mgate-agent/agent.yaml
```
